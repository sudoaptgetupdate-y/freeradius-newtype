# คู่มือการใช้งานระบบ FreeRADIUS V3.0 - Multi-Tenant (SaaS) Edition

เอกสารฉบับนี้รวบรวมฟังก์ชันการทำงานของระบบ FreeRADIUS รูปแบบ **คลาวด์แยกผู้เช่า (Multi-Tenancy / SaaS)** โดยแบ่งการใช้งานออกเป็น 2 ระดับหลัก คือ **ระดับเจ้าของระบบ (Master/Super Admin)** และ **ระดับลูกค้าผู้เช่า (Tenant Admin)**

---

## 🌟 ภาคที่ 1: ระบบจัดการสำหรับเจ้าของโปรเจ็ค (Master Administration)
*(สำหรับผู้ถือสิทธิ์ Super Admin เพื่อบริหารจัดการ Server และลูกค้าแต่ละราย)*

### 1.1 การบริหารจัดการลูกค้า (Tenant Management)
- **ภาพรวม:** แสดงรายชื่อลูกค้า (Site) ทั้งหมดที่อยู่ในระบบ พร้อมสถานะ (Active, Suspended, Expired)
- **สร้าง Site ใหม่ (Create Tenant):** ระบุชื่อบริษัท, สร้างบัญชีแอดมินให้ลูกค้า, และกำหนดโควต้า
- **ระงับ/ลบ Site:** สั่ง Suspend เพื่อระงับการล็อกอินของลูกค้า หรือ Delete ลบข้อมูลทิ้ง
- **โควต้าทรัพยากร:** จำกัดจำนวน User สูงสุด และจำนวน Mikrotik NAS ที่ลูกค้าใช้งานได้
- **สิทธิ์การเข้าถึง Log (Log Access):** สลับเปิด/ปิดสิทธิ์ให้ลูกค้าแต่ละรายดาวน์โหลด Log ย้อนหลัง 7 วันด้วยตัวเอง และระบบจัดการคำขอ (Approve/Reject) เมื่อลูกค้าขอดาวน์โหลด Log ย้อนหลัง 30-90 วัน
- **การสวมสิทธิ์ผู้ดูแลระบบผู้เช่า (Impersonation Mode):** ผู้ดูแลระบบระดับสูง (Super Admin) สามารถสวมสิทธิ์เป็นแอดมินของ Tenant นั้นๆ ได้ทันทีผ่านปุ่มล็อกอินสวมสิทธิ์ (Manage) โดยไม่ต้องกรอกรหัสผ่าน ซึ่งจะแสดงแถบแบนเนอร์แจ้งเตือนสถานะการสวมสิทธิ์ (Impersonation Banner) สีส้มด้านบนสุด และจำกัดขอบเขตการเข้าถึงเมนูรวมถึงฟังก์ชันต่างๆ เหมือน Tenant Admin ทราบและใช้งานจริง 100% พร้อมปุ่มกด "Exit" เพื่อดึงกลับสิทธิ์ Master Admin หลักได้ทันที


### 1.2 การตรวจสอบสถานะเซิร์ฟเวอร์ (Server Health & Monitoring)
- **System Dashboard:** ดูกราฟการใช้ทรัพยากร CPU, RAM, Disk Space และ Database Size
- **Global Online Users:** ดูจำนวนผู้ใช้งานอินเทอร์เน็ตที่กำลังออนไลน์จาก "ทุก Tenant รวมกัน"
- **Service Status:** ตรวจสอบและสั่ง Restart Service สำคัญเบื้องหลัง (`freeradius`, `postgresql`, `loki`, `vector`, `redis`)

### 1.3 การตั้งค่าระบบระดับกลาง (Global Configuration)
- **Global Settings (หน้าการตั้งค่าระบบหลัก):** แผงตั้งค่ารวมศูนย์สำหรับจัดการพารามิเตอร์หลักของระบบ โดยแอดมินสามารถกรอกรายละเอียดดังนี้:
  - **Telegram Bot Configuration:** ใส่ Bot Token, บัญชีผู้ใช้งานบอท (Bot ID/Username), Chat ID ส่วนกลางสำหรับส่งแจ้งเตือนระบบ (Server Alert) พร้อมสวิตช์ เปิด/ปิด
  - **Redis Connection:** ตั้งค่า IP Address/Host, Port และ Password ของเซิร์ฟเวอร์ Redis เพื่อใช้กับระบบคิวประมวลผลหลังบ้าน
  - **Loki & Vector Connection:** ระบุ Loki HTTP API URL และพอร์ต Syslog ของ Vector (สำหรับรับส่ง Log พ.ร.บ.)
  - **SMTP Server:** โฮสต์อีเมล SMTP, พอร์ต, บัญชีผู้ใช้, รหัสผ่าน และอีเมลผู้ส่ง สำหรับระบบแจ้งเตือนทางอีเมล
  - **General Settings:** เขตเวลา (Timezone) ของเซิร์ฟเวอร์
- **SSL / Certificate Management:** นำเข้า Public CA Certificate (เช่น Let's Encrypt) ลงใน FreeRADIUS EAP เพื่อรองรับความปลอดภัยของการเชื่อมต่อ 802.1X
- **Radius Dictionary Management:** อัปเดตชุดคำสั่ง Radius Attributes เพื่อรองรับอุปกรณ์เครือข่ายใหม่ๆ

### 1.4 การบริหารทีมงาน (Master Staff Management)
- สร้างบัญชีแอดมินระดับ Super Admin ให้กับทีมงาน (เช่น Support Team, Network Engineer)
- กำหนด Role ย่อยเพื่อจำกัดสิทธิ์ (เช่น ให้สิทธิ์ดูได้อย่างเดียว ห้ามลบ Site ลูกค้า)

### 1.5 การจัดการ Log ส่วนกลาง (Centralized Logs & Retention)
- ระบบเปลี่ยนมาใช้ **Vector** ร่วมกับ **Grafana Loki** แทนที่ระบบ Syslog เดิม เพื่อรองรับการเก็บข้อมูลพ.ร.บ. ขนาดใหญ่ระดับ 30GB+/วัน
- **Retention Policy:** ตั้งระบบล้างฐานข้อมูลอัตโนมัติใน Loki (เช่น ลบ Log เก่าเกิน 90 วัน) เพื่อคืนพื้นที่ Disk ให้เซิร์ฟเวอร์

### 1.6 Telegram Bot Integration (Master Level)
- **Telegram Settings:** มีหน้าต่างตั้งค่าสำหรับเชื่อมต่อ API ของ Telegram (ใส่ Bot Token) พร้อมสวิตช์เปิด/ปิดการทำงานของบอทส่วนกลาง
- **การแจ้งเตือน (Notifications):** แจ้งเตือนสถานะ Server ล่ม, ทรัพยากรเต็ม (CPU/RAM > 90%), และแจ้งเตือนเมื่อ Tenant ยื่นคำขอดาวน์โหลด Log (สามารถกด Approve จาก Telegram ได้เลย)
- **คำสั่งบอท (Commands):** พิมพ์ `/status` เพื่อดูสถานะเซิร์ฟเวอร์, พิมพ์ `/tenants` เพื่อดูจำนวน Site ที่แอคทีฟอยู่

## 🏢 ภาคที่ 2: ระบบจัดการสำหรับลูกค้าผู้เช่า (Tenant Administration)
*(สำหรับลูกค้าแต่ละ Site ใช้บริหารจัดการอินเทอร์เน็ตขององค์กรตนเอง ข้อมูลจะถูกแยกจาก Site อื่น 100%)*

### 2.1 Dashboard & Monitoring
- **Dashboard:** ดูจำนวนผู้ใช้งานออนไลน์ (Online Users) ภายใน Site ของตนเองแบบ Real-time
  - **Group Analytics Widget:** เพิ่มกราฟหรือวิดเจ็ตแสดงข้อมูลแบ่งกลุ่ม (เช่น Top Groups by Active Sessions, ปริมาณการใช้งานอินเทอร์เน็ตของแต่ละกลุ่ม เพื่อช่วยให้ Tenant Admin วิเคราะห์ Bandwidth และปรับแต่ง Profile ได้เหมาะสม)
- **Online Users:** ดู IP Address, ตรวจสอบระยะเวลาเชื่อมต่อ และสามารถสั่ง **"เตะ" (Disconnect)** ผู้ใช้งานออกจากระบบได้
- **History:** ตรวจสอบประวัติการเข้าใช้งาน, ปริมาณการดาวน์โหลด/อัปโหลด และสาเหตุที่ล็อกอินไม่สำเร็จ

### 2.2 Infrastructure & Packages
- **NAS (Network Access Server):** เพิ่มอุปกรณ์ Router/Firewall ของไซต์ตนเอง โดยต้องเลือกว่าเป็นยี่ห้อใด (เช่น Mikrotik, Fortigate, หรือ Other) เพื่อให้ระบบส่งค่าความเร็วและดึง Log ได้ถูกต้อง
- **Authentication Mode (802.1X vs Hotspot):** สามารถเปิด-ปิดโหมดการยืนยันตัวตนระดับองค์กร (WPA2/WPA3-Enterprise) เพื่อให้พนักงานกรอก Username/Password ฝังในเครื่องโดยไม่ต้องผ่านหน้าเว็บ หรือใช้งานควบคู่กับหน้า Hotspot ได้พร้อมกัน
- **Radius Profiles (Internet Packages):** สร้างแพ็กเกจอินเทอร์เน็ต โดยระบบออกแบบ UI/UX เป็น 3 โหมดเพื่อความยืดหยุ่น:
  - **1) MikroTik Template:** กรอกฟอร์มสำเร็จรูปสำหรับบีบความเร็ว (Rate-Limit), จำกัดเวลา (Session-Timeout), และจำกัดอุปกรณ์พร้อมกัน (Simultaneous-Use)
  - **2) Standard Enterprise Template (802.1X):** สำหรับอุปกรณ์ Enterprise (Cisco, Fortigate, Aruba) กรอกเฉพาะเลข VLAN ID ระบบจะแปลงเป็นมาตรฐาน RFC (Tunnel-Type, Tunnel-Private-Group-Id) ให้อัตโนมัติ
  - **3) Advanced Custom Mode:** ตารางเปล่าสำหรับกรอก VSA (Vendor-Specific Attributes) ด้วยตนเองเพื่อรองรับอุปกรณ์แปลกๆ แบบไม่จำกัดค่าย
- **MAC Bypass (MAB) / IoT Devices:** หน้าต่างสำหรับให้แอดมินนำ MAC Address ของอุปกรณ์ที่พิมพ์รหัสผ่านไม่ได้ (ปริ้นเตอร์, กล้องวงจรปิด) มาลงทะเบียนเพื่อให้อุปกรณ์เชื่อมต่อ 802.1X ผ่านเข้า VLAN ได้ทันที
- **Groups (Organizations):** จัดกลุ่มผู้ใช้งานภายใน Site ตามโครงสร้างองค์กร (เช่น แผนก, ชั้นปี, อาคาร) โดยแต่ละ Group จะผูกกับ Radius Profile (Internet Package) หนึ่งตัว ทำให้ผู้ใช้ทุกคนในกลุ่มได้รับความเร็วและ Policy เดียวกันโดยอัตโนมัติ:
  - **Profile Inheritance:** User ที่อยู่ใน Group จะได้รับ Profile จาก Group โดยอัตโนมัติ ไม่ต้องเลือก Profile เองทีละคน
  - **Status Inheritance:** หากกลุ่มถูกระงับการใช้งานทั้งหมด เมื่อมีการสร้างผู้ใช้ใหม่หรือย้ายผู้ใช้อื่นเข้ามาในกลุ่มนี้ ระบบจะกำหนดสถานะ Suspended ให้ผู้ใช้นั้นโดยอัตโนมัติ
  - **Group Actions Dropdown:** ปุ่มคำสั่งบริหารจัดการ (Transfer, Bulk Suspend, Reactivate, Delete) ถูกจัดเก็บในเมนู Dropdown (Action `...`) เพื่อความสะอาดตา โดยปุ่ม Reactivate All จะแสดงอัตโนมัติเฉพาะเมื่อกลุ่มมีสมาชิกที่ถูกระงับหรืออยู่ในถังขยะ
  - **Bulk Suspend:** ระงับการใช้งานของผู้ใช้ทุกคนใน Group พร้อมกัน พร้อมส่ง RADIUS CoA Disconnect-Request ไปยัง NAS ทันที
  - **Bulk Reactivate:** ปลดระงับการใช้งานและกู้คืน (Restore) ผู้ใช้จากถังขยะใน Group พร้อมกันเพื่อให้สามารถกลับมาล็อกอินได้
  - **Bulk Delete (Soft-Delete):** ลบบัญชีผู้ใช้ทุกคนใน Group โดยย้ายไปไว้ที่ถังขยะ (Trash Bin) ระบบจะส่ง CoA ก่อนลบเสมอ
  - **Group Details:** รายละเอียดเชิงลึกของแต่ละกลุ่ม แสดงเฉพาะข้อมูลสถิติและข้อมูลทั่วไป ได้แก่:
    - **ข้อมูลทั่วไป:** ชื่อกลุ่ม, รายละเอียด, Radius Profile ที่เชื่อมโยงอยู่ (แสดง Limits ความเร็ว/เวลา)
    - **สถิติสมาชิก:** จำนวนสมาชิกทั้งหมด (Total Members), สมาชิกออนไลน์ (Online Users), และสมาชิกที่ถูกระงับใช้งาน
  - **ตัวอย่างการใช้งาน:** โรงเรียนสร้าง Group "ม.6" ผูกกับ Profile "Student-10Mbps" เมื่อนักเรียนจบการศึกษา กด Bulk Suspend หรือ Bulk Delete ทั้ง Group ได้ในคลิกเดียว โดยข้อมูลประวัติการใช้เน็ตจะยังคงอยู่ครบถ้วน

### 2.3 User & Voucher Management
- **Users:** เพิ่ม, นำเข้า (Import CSV), แก้ไข, และระงับสิทธิ์ผู้ใช้งานอินเทอร์เน็ต
  - **การสร้าง User:** เลือก **Group** ที่ต้องการเข้าร่วม (แทนการเลือก Profile โดยตรง) ระบบจะ inherit Profile ของ Group นั้นให้อัตโนมัติ
  - **การย้าย Group:** แก้ไข User แล้วเปลี่ยน Group → ระบบอัปเดต Profile ใน `radusergroup` และส่ง CoA เพื่อใช้ Policy ใหม่ทันทีโดยไม่ต้อง Reconnect ด้วยตนเอง
  - **การทำงานของสถานะบัญชี (User Account Statuses):**
    - **1) Active (ปกติ):** บัญชีผู้ใช้งานปกติ สามารถเชื่อมต่ออินเทอร์เน็ตได้
    - **2) Suspended (ระงับชั่วคราว):** มีการสร้างเรคคอร์ด `Auth-Type = Reject` ใน `radcheck` เพื่อระงับการล็อกอินชั่วคราวโดยยังคงเห็นข้อมูลในตาราง Active Users และสามารถกดปลดระงับได้ทันที
    - **3) Soft-Deleted (ถังขยะ - Trash Bin):** ผู้ใช้งานที่ถูกลบจะถูกเปลี่ยนสถานะในฟิลด์ `deletedAt` และถูกย้ายจากรายชื่อปกติไปยังแท็บ **Trash Bin** เพื่อป้องการลบผิดพลาดและคงข้อมูลประวัติการเชื่อมต่อ (radacct) ไว้ตาม พ.ร.บ. โดยระยะเวลาการจัดเก็บในถังขยะสามารถตั้งค่าได้ในระบบ (Trash Retention Days) และระบบ Cleanup Service หลังบ้านจะเคลียร์ออกถาวร (Hard Delete) เมื่อพ้นกำหนดเวลา
  - **User Details (หน้ารายละเอียดผู้ใช้):** คลิกเข้าดูข้อมูลเชิงลึกของผู้ใช้แต่ละราย ได้แก่:
    - **ข้อมูลพื้นฐาน:** Username, ชื่อ-นามสกุล, อีเมล, เบอร์โทร, Group ที่สังกัด, Package/Profile ปัจจุบัน, สถานะบัญชี
    - **สถานะ Session ปัจจุบัน:** IP Address, MAC Address, อุปกรณ์ที่ใช้งาน, ระยะเวลาออนไลน์, และปริมาณ Data ที่ใช้ไปในช่วงนั้น
    - **โควต้า & แพ็กเกจ:** ข้อมูลวันหมดอายุ (Expiry), Session-Timeout, และ Data Quota ที่เหลืออยู่
    - **ประวัติการเชื่อมต่อ (Session History):** ตารางรายการ radacct ย้อนหลัง แสดง IP, MAC, เวลาเข้า-ออก, ปริมาณการใช้งาน (Upload/Download)
    - **การดำเนินการ:** ปุ่ม Disconnect (CoA), Suspend/Reactivate, Change Group, Reset Password, Delete User, Restore (กรณีอยู่ในถังขยะ), Permanent Delete (กรณีลบจากถังขยะถาวร)
- **Vouchers:** สร้างแพ็กเกจบัตรอินเทอร์เน็ต (Voucher Packages) และสั่งเจนเนอเรตรหัสบัตร (Voucher Batches) พร้อม Export ไปสั่งพิมพ์
  > ⚠️ Voucher ผูกกับ **Profile โดยตรง** (ไม่ผ่าน Group) เนื่องจาก Voucher เป็นแบบ Stateless ไม่มีความเป็นสมาชิกถาวรขององค์กร

### 2.4 Auxiliary Router API (เฉพาะ Mikrotik)
- **Settings:** เชื่อมต่อ API ของ Mikrotik ตัวเองเข้าระบบ พร้อมกด Test Connection
- **Walled Garden & IP Bindings:** ตั้งค่าอนุญาตเว็บไซต์ที่เข้าได้ฟรีก่อนล็อกอิน, จัดการอุปกรณ์ที่ไม่ต้องล็อกอิน (Bypass MAC Address) หรือแบนอุปกรณ์ชั่วคราว
- **NAS Monitoring:** ดูสถานะการใช้ CPU, อุณหภูมิ, และการแจก IP ภายใน Router ของสาขาตนเอง

### 2.5 Captive Portal & Advertisements (การปรับแต่ง)
- **Portal Customization & Settings:** แผงตั้งค่าหน้ารวมล็อกอินและลงทะเบียนของ Tenant แต่ละแห่ง (ตาราง `tenant_portal_settings`) เพื่อแยกภาพลักษณ์แบรนด์ (Tenant Branding) ได้แก่:
  - **Logo & Info:** อัปโหลดโลโก้หน่วยงาน, กำหนดชื่อแสดงผล (Org Name), และข้อความหมายเหตุแถบล่างหน้าจอ (Footer Note)
  - **Theme Customization:** กำหนดรหัสสีธีม (Primary Color) สำหรับควบคุมหน้าตาของปุ่มกดและฟังก์ชันหน้าเว็บ
  - **Terms of Service (TOS):** แอดมินสามารถอัปเดตข้อตกลงและเงื่อนไขการใช้งาน (HTML/Markdown) เพื่อบังคับให้ผู้ใช้งานกดยอมรับก่อนเข้าเล่นเน็ตหรือลงทะเบียน
- **External / Social Login:** เปิด-ปิด ฟังก์ชันลงชื่อเข้าใช้งานด้วยเครือข่ายโซเชียล โดยระบบรองรับบริการภายนอกที่ไม่มีค่าใช้จ่ายแอบแฝง:
  - **Google Login:** ตรวจสอบและลงทะเบียนเข้าเน็ตอัตโนมัติด้วย Gmail/Google API
  - **LINE Login:** ยืนยันตัวตนผ่าน LINE Developers รองรับระบบ Auto-Login ไร้รอยต่อหากเปิดเว็บในแอปพลิเคชัน LINE
- **Self-Registration (สมัครสมาชิก):** เปิด-ปิด หน้าสมัครเข้าใช้เน็ตด้วยตนเองผ่านแบบฟอร์มหน้าเว็บ โดยผู้ใช้งานที่สมัครจะถูกผูกเข้ากับ `defaultRegisterProfile` ในตาราง `radusergroup` หลังบ้านโดยอัตโนมัติ
- **Advertisement:** สร้างโฆษณาก่อนเข้าใช้งานเน็ต กำหนดเวลา Countdown

### 2.6 Security & Admin Settings
- **Administrators:** สร้างบัญชีผู้ดูแลระบบย่อยภายใน Site ของตนเอง (เช่น พนักงานต้อนรับ, ช่างไอที)
- **Account Settings:** เปลี่ยนรหัสผ่านของแอดมิน
- **Log Management:** บริหารจัดการ Log พ.ร.บ. คอมพิวเตอร์ ของ Router แต่ละตัวผ่านระบบ Grafana Loki ส่วนกลาง
  - ดึงข้อมูลย้อนหลัง 7 วันได้ด้วยตนเอง (หากส่วนกลางเปิดสิทธิ์ไว้ให้)
  - ส่งคำร้อง (Request Export) เพื่อขอดาวน์โหลดข้อมูลย้อนหลัง 30-90 วัน (ต้องรอส่วนกลางอนุมัติ)

### 2.7 Telegram Bot Integration (Tenant Level)
- **Telegram Settings:** มีเมนูสำหรับใส่ Chat ID (หรือ Group ID ของร้าน) พร้อมสวิตช์สำหรับเปิด/ปิดฟังก์ชันแจ้งเตือนผ่าน Telegram แบบราย Site ได้ด้วยตนเอง
- **Guest Approval via Telegram:** เมื่อมีแขกสมัครใช้งานอินเทอร์เน็ตหน้าเว็บ (Captive Portal) ระบบจะส่งข้อความแจ้งเตือนเข้า Telegram ชาแนลของร้าน พร้อมปุ่มกด `[Approve]` และ `[Decline]` แบบ Inline Keyboard
- **NAS Alerts:** แจ้งเตือนเมื่อ Router Mikrotik ของสาขาออฟไลน์ (เน็ตตัด/ไฟดับ)
- **คำสั่งบอท (Commands):** สามารถพิมพ์สั่งงานระบบผ่านแชท Telegram ได้ทันที ดังนี้:
  - `/status` - เช็กเราเตอร์หลัก สถานะระบบโดยรวม พร้อมปุ่ม Inline: `[ดึงผู้ใช้ออนไลน์]`, `[🔄 PING Gateway]`
  - `/user <username>` - เรียกดูการ์ดโปรไฟล์ของผู้ใช้นั้นๆ แสดงชื่อ Group ที่สังกัด พร้อมปุ่ม Inline:
    - `[⚡ เตะอุปกรณ์ (Kick)]` (สำหรับลบ Active Session ป้องกันการค้าง)
    - `[🚫 ระงับ (Suspend)]` / `[🟢 ปลดระงับ (Reactivate)]`
    - `[➕ ขยายแพ็กเกจ (Extend)]`
    - `[🗑️ ลบบัญชี (Soft-Delete)]` (ย้ายผู้ใช้ลงถังขยะ Trash Bin)
    - `[♻️ กู้คืนบัญชี (Restore)]` (กรณีบัญชีอยู่ในถังขยะ)
  - `/group <groupname>` - แสดงการ์ดรายละเอียดของกลุ่ม เช่น จำนวนสมาชิกออนไลน์, โปรไฟล์ที่ผูกอยู่ พร้อมปุ่ม Inline:
    - `[🚫 ระงับการใช้งานกลุ่ม (Suspend Group)]`
    - `[🟢 ปลดระงับกลุ่ม (Reactivate Group)]`
    - `[🗑️ ลบสมาชิกทั้งกลุ่ม (Soft-Delete Group Users)]` (ย้ายผู้ใช้ทั้งหมดในกลุ่มลงถังขยะ)

  - `/find <ชื่อ/เบอร์โทร/อีเมล>` - ค้นหาผู้ใช้งานระบบ และส่งคืนลิสต์ปุ่มผู้ใช้เพื่อกดเปิดดูการ์ดข้อมูลส่วนตัว
  - `/voucher <จำนวน> <แพ็กเกจ>` - สั่งเจนบัตรคูปองเน็ตแบบด่วนแล้วรับปุ่มกดดาวน์โหลดไฟล์ PDF หรือแสดงผลรหัสในแชท

## 📱 ภาคที่ 3: ระบบจัดการสำหรับผู้ใช้งานอินเทอร์เน็ต (Self-Care User Portal)
*(สำหรับพนักงาน หรือแขกที่ใช้งานอินเทอร์เน็ต เพื่อจัดการบัญชีของตนเอง ลดภาระงานของแอดมิน)*

### 3.1 Account & Dashboard
- **Profile:** เปลี่ยนรหัสผ่าน (Change Password) และแก้ไขข้อมูลส่วนตัว
- **Quota Check:** ตรวจสอบแพ็กเกจปัจจุบัน ดูวันหมดอายุ และปริมาณ Data/Time ที่เหลืออยู่
- **History:** ตรวจสอบประวัติการเข้าใช้งานของตนเอง (เพื่อเช็กว่ามีคนอื่นเอารหัสไปแอบใช้หรือไม่)

### 3.2 Device & Session Management (การจัดการอุปกรณ์)
- **Active Sessions:** ดูรายการอุปกรณ์ (MAC Address) ที่กำลังล็อกอินด้วยบัญชีนี้อยู่
- **Self-Disconnect (Clear MAC):** ปุ่ม **"เตะตัวเอง"** ใช้แก้ปัญหาเน็ตหลุดแต่ Session ค้างในระบบ ทำให้ไปล็อกอินเครื่องอื่นแล้วติด Error "Limit Reached" (ระบบจะส่งคำสั่ง **RADIUS CoA** ไปเตะอุปกรณ์ที่ค้างอยู่ให้อัตโนมัติ)
- **Device Management:** สำหรับผู้ที่ใช้ 802.1X สามารถลบอุปกรณ์ (Device) เก่าทิ้ง เพื่อนำโควต้าไปให้เครื่องใหม่เชื่อมต่อแทนได้
